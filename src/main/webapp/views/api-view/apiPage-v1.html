Web API

The PRIDE Cluster RESTful APIs provide programmatic access to read our cluster data. Query a particular peptide, gather general statistics, read cluster details, and more.

Web browsable API

The PRIDE Cluster RESTful API is web browsable, which means that:

  • The query results returned by the API are available in JSON format. This ensures that they can be viewed by human and accessed programmatically by computer.
  • The main RESTful API page provides a simple web-based user interface, which allows developers can familiarise themselves with the API and get a better sense of the PRIDE Cluster data before writing single line of code.

Versioning

To ensure that changes in the PRIDE Cluster API don't break the applications relying on it, the API is versioned, and the version is included in the API's URL.

For example, the current API is at Version 1 and is available at http://www.ebi.ac.uk/pride/ws/cluster/v1, and the next version will be available at http://www.ebi.ac.uk/pride/ws/cluster/v2.

The latest version of the API can be accessed at http://www.ebi.ac.uk/pride/ws/cluster/current, but it's not recommended to use this endpoint for long-term development as the underlying data model may change.

No backward-incompatible changes are made to each version after it's been made public. More specifically, it's guaranteed that within one version there will be no:

  • changing urls
  • deleting or renaming data fields
  • changing data field types

The following non-disruptive changes may be implemented to a public API:

  • adding new endpoints
  • adding new data fields
  • adding new filtering methods

An advance notice will be given before obsoleting an API version. To stay up to date, please consider signing up for the PRIDE Twitter account.

API version 1 documentation

The PRIDE Cluster RESTful API contains the following URL main endpoints:

  • project: provide the details of all original projects where a cluster's spectra and peptide identifications come from.
  • cluster: main endpoint for retrieving the details related to a cluster, such as: species, modifications, and peptides.
  • stats: main endpoint for retrieving general statistics on the whole of the PRIDE cluster database.
  • psm: provide the details of all peptide-spectrum-match within a cluster
  • library: details on the spectrum libraries.

Search peptide

To search peptide, you can using the following web service endpoint.

  • /cluster/list

This web service endpoint provide a number of options to help you navigate the results:

  • Filtering: Filter the results using modification name (modFilters) or species name (speciesFilters).
  • Pagination: If there are too many resutls, it is possible to paginate the results (page).
  • Sort: The results can be sorted using the number of spectra, the number of projects and the ratio of the peptide identifications within a cluster.
  • Facets: Facets can be retrieved for both species and modifications which show the result counts.
