PRIDE Cluster pipeline

What is the PRIDE Cluster pipeline?

The PRIDE Cluster pipeline is responsible for extracting spectra and their related metadata from PRIDE Archive, and then feed these information into our clustering pipeline, the clustering pipeline is going to group similar spectra into the clusters.

What are main steps of the pipeline?

The pipeline consist of the following steps (illustrated below):

  • Filter spectra from PRIDE Archive, remove bad quality spectra, and then annotation each spectra with metadata, such as species, modifications and peptides.
  • Move the filtered spectra in the format of MGF files into the Hadoop Cluster.
  • Using Map-Reduce to group the spectra into clusters.
  • Move the results out of the Hadoop cluster.
  • Quality check of the results.
  • Load the cluster results into the production database for the new release.
  • Generate spectrum libraries using the clustering results.

Why Hadoop?

The original PRIDE Cluster is based on Load Sharing Facility (LSF), this pipeline has encountered scalability issues due the fast data increase from PRIDE Archive.

We set to improve the scalability and performance of the pipeline, Hadoop Map-Reduce is natural fit as it provides a mass parallel computing framework.
